Request an API Key

All of my API's are available for free to any interested developer. The Elastic Search API specifically can easily have a very high load with large complex queries, so please be considerate when providing tools that use this API.

Currently only the Shop Indexer ElasticSearch API requires an API Key. If you would just like to take the ES API for a test drive before determining whether or not to use it, you may temporarily use the Public Development API Key. Please note this API Key is heavily rate limited and is not suitable for any public tool. If you run into rate limits while developing a tool, I highly recommend you get our own API key!


    Development API Key Header:

    Authorization: DEVELOPMENT-Indexer


Why do I need an API Key?

I would like to keep track of who is using this API extensively so that I can communicate with tool developers in the event that my server starts to get overloaded or if I find queries that are particularly burdensome. API Keys allow me to do this easily, as well as report on how much your application is being used.

How do I use the API Key?

Simply embed the API Key into an "Authorization" header in your HTML requests. You will receive JSON error responses if the key is invalid, being rate limited, etc.

What are the Rate Limits?

The default rate limits for public applications is 500 requests every 60s. If your application sees heavy use, I may be able to increase that, however this is already a pretty high bar!

Note: You do not have to request an API Key for development!

I want it to be as easy as possible for people to experiment with developing tools using this index. Please feel free to use the Development API Key until a point where your site or tool has hundreds of active users. There is no need to request an API Key just to experiment or write your own personal simple tool as long as it does not make tens of requests a second repeatedly.
